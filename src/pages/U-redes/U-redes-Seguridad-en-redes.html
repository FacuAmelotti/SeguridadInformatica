¿Qué es la seguridad en redes?

  • ● La seguridad en redes comprende políticas y prácticas adoptadas para prevenir y supervisar el acceso no autorizado, el uso indebido, la modificación o la denegación de una red informática.
  • ● Protege tanto los recursos como la información en la red.
  • ● Abarca hardware, software y políticas de seguridad.
  • ● Es un componente crítico en cualquier infraestructura de redes moderna.
nmap -sV 192.168.1.1 #Escanea un dispositivo para identificar servicios y versiones expuestas.

La seguridad en redes es como el sistema de seguridad de una casa: necesitas cerraduras en las puertas (firewalls), cámaras de vigilancia (IDS/IPS), y cajas fuertes para tus objetos de valor (cifrado).

Conceptos básicos de seguridad

CIA (Confidencialidad, Integridad, Disponibilidad)

  • ● Confidencialidad: Garantiza que la información es accesible solo para aquellos autorizados.
  • ● Integridad: Asegura que la información no ha sido alterada durante su transmisión o almacenamiento.
  • ● Disponibilidad: Asegura que la información y los recursos estén disponibles cuando se necesiten.

Amenazas y vulnerabilidades

  • ● Amenaza: Cualquier circunstancia o evento que pueda causar daño a un sistema.
  • ● Vulnerabilidad: Debilidad en un sistema que puede ser explotada por una amenaza.
  • ● Riesgo: Probabilidad de que una amenaza explote una vulnerabilidad.
nikto -h 192.168.1.100 #Herramienta para escanear vulnerabilidades en servidores web.

Tipos de ataques comunes

Malware

  • ● Software malicioso diseñado para dañar o infiltrarse en sistemas.
  • ● Incluye virus, gusanos, troyanos, ransomware, spyware, etc.
  • ● Se propaga a través de archivos adjuntos, descargas infectadas o dispositivos comprometidos.

Phishing

  • ● Técnica para obtener información confidencial haciéndose pasar por una entidad legítima.
  • ● Utiliza principalmente correos electrónicos o sitios web falsos.
  • ● Busca robar credenciales, datos bancarios o información personal.
whois suspicious-domain.com #Verifica la información de registro de un dominio sospechoso.

DDoS (Denegación de Servicio Distribuida)

  • ● Ataque que busca hacer que un servicio o recurso sea inaccesible.
  • ● Utiliza múltiples sistemas (generalmente una botnet) para inundar el objetivo con tráfico.
  • ● Afecta la disponibilidad de servicios y sitios web.

Man-in-the-Middle

  • ● Atacante intercepta comunicaciones entre dos partes sin que estas lo sepan.
  • ● Puede leer, modificar o inyectar nuevos datos en la comunicación.
  • ● Común en redes Wi-Fi no seguras o mal configuradas.
wireshark #Herramienta para capturar y analizar tráfico de red (para fines educativos).

Mecanismos de protección

Firewalls

  • ● Sistema que monitorea y filtra el tráfico de red según reglas preestablecidas.
  • ● Actúa como barrera entre redes confiables y no confiables.
  • ● Pueden ser basados en hardware, software o ambos.
  • ● Tipos: Stateless, Stateful, Next-Generation Firewalls (NGFW).
sudo iptables -L #Visualiza las reglas del firewall en sistemas Linux.

IDS/IPS (Sistemas de Detección/Prevención de Intrusiones)

  • ● IDS: Monitorea la red y alerta sobre actividades sospechosas.
  • ● IPS: Además de detectar, toma medidas activas para bloquear amenazas.
  • ● Pueden basarse en firmas (patrones conocidos) o en comportamientos anómalos.

Cifrado y VPN

Cifrado simétrico vs asimétrico

  • ● Simétrico: Usa la misma clave para cifrar y descifrar (AES, DES).
  • ● Asimétrico: Usa par de claves pública/privada (RSA, ECC).
  • ● A menudo se usan juntos: asimétrico para intercambiar claves y simétrico para cifrar datos por eficiencia.

SSL/TLS

  • ● Protocolos que proporcionan comunicaciones seguras en redes (especialmente en HTTPS).
  • ● Garantizan confidencialidad, integridad y autenticación.
  • ● Utilizan certificados digitales para verificar la identidad de los servidores.
openssl s_client -connect example.com:443 #Verifica la configuración SSL/TLS de un sitio web.

Redes privadas virtuales (VPN)

  • ● Crean una conexión cifrada (túnel) a través de redes no seguras.
  • ● Permiten acceso remoto seguro a recursos privados.
  • ● Ocultan la IP real del usuario y cifran el tráfico.
  • ● Protocolos comunes: OpenVPN, IPsec, WireGuard.

Principios de hardening

  • ● Defensa en profundidad: Implementar múltiples capas de seguridad.
  • ● Principio de menor privilegio: Otorgar solo los permisos mínimos necesarios.
  • ● Actualizaciones regulares: Mantener sistemas y software actualizados.
  • ● Configuración segura: Deshabilitar servicios innecesarios y configurar correctamente los necesarios.
  • ● Monitoreo continuo: Revisar registros y alertas de seguridad.
sudo apt update && sudo apt upgrade #Actualiza el sistema para corregir vulnerabilidades conocidas.

Recuerda: La seguridad no es un producto, sino un proceso continuo. Ningún sistema es 100% seguro, pero podemos implementar medidas para minimizar los riesgos.
