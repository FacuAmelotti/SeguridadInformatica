¿Qué es la seguridad en redes?

  • ● La seguridad en redes comprende políticas y prácticas adoptadas para prevenir y supervisar el acceso no autorizado, el uso indebido, la modificación o la denegación de una red informática.
  • ● Protege tanto los recursos como la información en la red.
  • ● Abarca hardware, software y políticas de seguridad.
  • ● Es un componente crítico en cualquier infraestructura de redes moderna.
nmap -sV 192.168.1.1 #Escanea un dispositivo para identificar servicios y versiones expuestas.
netstat -tuln #Muestra los puertos que están escuchando en el sistema y los procesos asociados.

La seguridad en redes es como el sistema de seguridad de una casa: necesitas cerraduras en las puertas (firewalls), cámaras de vigilancia (IDS/IPS), y cajas fuertes para tus objetos de valor (cifrado). Ninguna medida por sí sola es suficiente; es la combinación de todas ellas lo que proporciona una protección efectiva.

Conceptos básicos de seguridad

CIA (Confidencialidad, Integridad, Disponibilidad)

Confidencialidad

  • Garantiza que la información es accesible solo para aquellos autorizados a tener acceso.
  • Se implementa mediante:
    • Cifrado de datos (en reposo y en tránsito)
    • Control de acceso y autenticación
    • Clasificación de la información
    • Políticas de seguridad
  • Un fallo en la confidencialidad resulta en divulgación no autorizada de información.

Integridad

  • Asegura que la información no ha sido alterada durante su transmisión o almacenamiento.
  • Se implementa mediante:
    • Funciones hash (SHA-256, MD5)
    • Firmas digitales
    • Control de versiones
    • Checksums
  • Un fallo en la integridad resulta en modificación no autorizada de datos.

Disponibilidad

  • Asegura que la información y los recursos estén disponibles cuando se necesiten.
  • Se implementa mediante:
    • Redundancia de sistemas
    • Copias de seguridad
    • Planes de recuperación ante desastres
    • Balanceo de carga
    • Protección contra DDoS
  • Un fallo en la disponibilidad resulta en interrupción del servicio o acceso a los recursos.
sha256sum archivo.txt #Calcula el hash SHA-256 de un archivo para verificar su integridad.

Amenazas y vulnerabilidades

Amenaza

  • Cualquier circunstancia o evento que pueda causar daño a un sistema.
  • Tipos de amenazas:
    • Naturales: terremotos, inundaciones, incendios
    • Humanas: hackers, usuarios maliciosos, errores de usuarios
    • Ambientales: fallos eléctricos, picos de tensión
    • Técnicas: malware, exploits, fallos de software
  • Actores de amenaza: script kiddies, hacktivistas, ciberdelincuentes, estados-nación, insiders

Vulnerabilidad

  • Debilidad en un sistema que puede ser explotada por una amenaza.
  • Tipos de vulnerabilidades:
    • Técnicas: errores de software, configuraciones incorrectas, parches faltantes
    • Físicas: acceso no controlado, falta de seguridad física
    • Humanas: susceptibilidad a ingeniería social, falta de concientización
    • Organizacionales: procesos deficientes, políticas inadecuadas
  • Las vulnerabilidades se catalogan en bases de datos como CVE (Common Vulnerabilities and Exposures)

Riesgo

  • Probabilidad de que una amenaza explote una vulnerabilidad y el impacto resultante.
  • Fórmula: Riesgo = Probabilidad × Impacto
  • Gestión de riesgos:
    • Identificación: encontrar y catalogar riesgos
    • Evaluación: analizar probabilidad e impacto
    • Tratamiento: aceptar, transferir, mitigar o evitar
    • Monitoreo: seguimiento continuo
nikto -h 192.168.1.100 #Herramienta para escanear vulnerabilidades en servidores web.
searchsploit apache 2.4 #Busca vulnerabilidades conocidas para una versión específica de software.

Tipos de ataques comunes

Malware

Tipos de malware

  • Virus: Código malicioso que se adjunta a archivos legítimos y requiere acción humana para propagarse.
  • Gusanos: Se propagan automáticamente sin intervención humana a través de redes.
  • Troyanos: Software malicioso disfrazado como programa legítimo.
  • Ransomware: Cifra los datos del usuario y exige un rescate para recuperarlos.
  • Spyware: Recopila información del usuario sin su conocimiento.
  • Adware: Muestra publicidad no deseada.
  • Rootkits: Oculta su presencia y permite acceso privilegiado al sistema.
  • Keyloggers: Registra las pulsaciones de teclas para robar contraseñas.
  • Botnets: Redes de dispositivos infectados controlados remotamente.

Vectores de infección

  • Correos electrónicos con archivos adjuntos maliciosos
  • Descargas de sitios web no confiables
  • Dispositivos USB infectados
  • Vulnerabilidades no parcheadas en software
  • Scripts maliciosos en sitios web (drive-by downloads)
  • Aplicaciones falsas o modificadas
  • Ingeniería social para engañar a los usuarios

Protección contra malware

  • Software antivirus/antimalware actualizado
  • Mantener sistemas operativos y aplicaciones actualizados
  • Filtrado de correo electrónico
  • Navegación segura (HTTPS)
  • Principio de menor privilegio (usuarios sin derechos administrativos)
  • Sandboxing (entornos aislados para ejecutar código sospechoso)
  • Respaldos regulares (3-2-1: 3 copias, 2 medios diferentes, 1 fuera del sitio)
  • Educación de usuarios
clamscan -r /ruta/a/escanear #Escanea recursivamente un directorio en busca de malware usando ClamAV.

Phishing

Tipos de phishing

  • Phishing genérico: Ataques masivos no personalizados.
  • Spear phishing: Ataques dirigidos a personas u organizaciones específicas.
  • Whaling: Dirigido a ejecutivos de alto nivel.
  • Smishing: Phishing a través de mensajes SMS.
  • Vishing: Phishing mediante llamadas telefónicas.
  • Clone phishing: Replica mensajes legítimos con enlaces maliciosos.
  • Pharming: Redirige tráfico de sitios legítimos a sitios falsos.

Indicadores de phishing

  • Errores gramaticales u ortográficos
  • Dominios sospechosos o similares a los legítimos
  • URLs que no coinciden con el supuesto remitente
  • Solicitudes urgentes o amenazadoras
  • Peticiones de información sensible
  • Ofertas demasiado buenas para ser verdad
  • Remitentes desconocidos o inesperados
  • Correos genéricos sin personalización

Protección contra phishing

  • Filtros anti-spam y anti-phishing
  • Verificación de remitentes (SPF, DKIM, DMARC)
  • Autenticación multifactor (MFA/2FA)
  • Educación y concientización de usuarios
  • Simulaciones de phishing para entrenamiento
  • Verificar manualmente las URLs antes de hacer clic
  • No proporcionar información sensible por correo o teléfono
  • Usar gestores de contraseñas para evitar introducirlas en sitios falsos
whois suspicious-domain.com #Verifica la información de registro de un dominio sospechoso.
curl -I https://example.com #Verifica los encabezados HTTP de un sitio para buscar indicadores sospechosos.

DDoS (Denegación de Servicio Distribuida)

Tipos de ataques DDoS

  • Ataques volumétricos: Inundan la red con tráfico (UDP floods, ICMP floods).
  • Ataques de protocolo: Explotan vulnerabilidades en protocolos (SYN floods, Ping of Death).
  • Ataques de capa de aplicación: Dirigidos a aplicaciones web (HTTP floods, Slowloris).
  • Ataques de amplificación: Utilizan servidores legítimos para multiplicar el tráfico (NTP, DNS, SSDP).
  • TCP state-exhaustion: Agotan los recursos de conexión de dispositivos de red.
  • Ataques de fragmentación: Envían paquetes fragmentados malformados.

Impacto de los ataques DDoS

  • Interrupción de servicios en línea
  • Pérdidas financieras (ventas, productividad)
  • Daño a la reputación de la marca
  • Costos adicionales de infraestructura
  • Posible extorsión (DDoS como amenaza)
  • Violación de SLAs (acuerdos de nivel de servicio)
  • Distracción para ocultar otros ataques

Mitigación de DDoS

  • Sobreaprovisionamiento de recursos (ancho de banda, servidores)
  • Servicios de protección DDoS (Cloudflare, Akamai, AWS Shield)
  • Filtrado de tráfico y listas negras
  • Balanceadores de carga
  • Sistemas de detección de anomalías
  • Content Delivery Networks (CDNs)
  • Configuración de rate limiting
  • Planificación de respuesta a incidentes
netstat -ant | grep SYN_RECV | wc -l #Cuenta las conexiones TCP en estado SYN_RECV, útil para detectar posibles SYN floods.

Man-in-the-Middle

Técnicas de ataque MitM

  • ARP spoofing: Falsificación de tablas ARP para redirigir tráfico.
  • DNS spoofing: Falsificación de respuestas DNS.
  • HTTPS spoofing: Interceptación de conexiones HTTPS con certificados falsos.
  • SSL stripping: Degradación de conexiones HTTPS a HTTP.
  • Wi-Fi eavesdropping: Interceptación de tráfico en redes inalámbricas.
  • Evil twin: Puntos de acceso Wi-Fi maliciosos que imitan redes legítimas.
  • Session hijacking: Robo de cookies o tokens de sesión.

Escenarios comunes

  • Redes Wi-Fi públicas no seguras
  • Redes corporativas comprometidas
  • Dispositivos de red no protegidos
  • Comunicaciones inseguras cliente-servidor
  • Conexiones a puntos de acceso falsos
  • Interceptaciones físicas de líneas de comunicación

Prevención de ataques MitM

  • HTTPS (TLS/SSL) en todas las comunicaciones
  • Certificate Pinning (fijación de certificados)
  • VPNs para cifrar tráfico
  • Verificación de certificados
  • DNSSEC para proteger consultas DNS
  • Detección de ARP spoofing
  • Evitar redes Wi-Fi públicas o usar VPN al conectarse
  • Autenticación mutua (2-way SSL)
wireshark #Herramienta para capturar y analizar tráfico de red (para fines educativos).
arp -a #Muestra la tabla ARP para detectar posibles duplicados sospechosos.

Fuerza bruta

Tipos de ataques de fuerza bruta

  • Fuerza bruta simple: Prueba todas las combinaciones posibles.
  • Ataque de diccionario: Usa listas de palabras comunes.
  • Ataque híbrido: Combina diccionarios con variaciones algorítmicas.
  • Rainbow tables: Utiliza tablas precalculadas de hashes.
  • Credential stuffing: Prueba credenciales filtradas en diferentes servicios.
  • Password spraying: Prueba pocas contraseñas comunes en muchas cuentas.

Objetivos comunes

  • Formularios de inicio de sesión web
  • Servicios SSH, FTP, RDP
  • Protocolos de correo electrónico (SMTP, POP3, IMAP)
  • Cifrado de archivos (ZIP, RAR protegidos)
  • Sistemas de gestión de contenido (WordPress, Joomla)
  • Paneles de administración
  • APIs y servicios web

Defensas contra fuerza bruta

  • Políticas de contraseñas robustas
  • Bloqueo de cuentas después de varios intentos fallidos
  • Captchas para prevenir automatización
  • Autenticación de dos factores (2FA/MFA)
  • Retrasos incrementales entre intentos fallidos
  • Registro y alertas de intentos fallidos
  • Cambio regular de contraseñas
  • Filtros de IP y geolocalización
  • Implementación de mecanismos de sal y pimentado (salt & pepper) en contraseñas almacenadas
fail2ban-client status sshd #Muestra las IPs bloqueadas por intentos fallidos de acceso SSH.
hydra -V -f -l usuario -P wordlist.txt ssh://192.168.1.10 #Herramienta de auditoría para probar la resistencia contra ataques de fuerza bruta.

Mecanismos de protección

Firewalls

Tipos de firewalls

  • Firewalls de filtrado de paquetes: Operan en capa 3-4, filtran basándose en IPs, puertos y protocolos.
  • Firewalls con estado (stateful): Mantienen registro del estado de las conexiones.
  • Firewalls de aplicación (proxy): Operan en capa 7, analizan tráfico específico de aplicaciones.
  • Next-Generation Firewalls (NGFW): Combinan funciones tradicionales con IPS y filtrado avanzado.
  • Firewalls basados en host: Se ejecutan en servidores individuales o endpoints.
  • Firewalls basados en hardware: Dispositivos físicos dedicados.
  • Firewalls basados en software: Soluciones implementadas como software.
  • Firewalls basados en la nube: Servicios de firewall ofrecidos como SaaS.

Implementación y arquitectura

  • DMZ (Zona desmilitarizada): Segmento de red entre la red interna y externa.
  • Arquitectura de firewall único: Un firewall protege todo el perímetro.
  • Arquitectura multi-firewall: Varios firewalls para diferentes segmentos de red.
  • Firewall a nivel de red: Protege toda la infraestructura de red.
  • Firewall a nivel de servidor: Protege servidores específicos.
  • Firewall a nivel de aplicación: Protege aplicaciones específicas.
  • Defensa en profundidad: Múltiples capas de firewalls y seguridad.

Configuración de reglas

  • Principio de denegación por defecto: Bloquear todo tráfico excepto el explícitamente permitido.
  • Reglas de entrada/salida: Control bidireccional del tráfico.
  • Filtrado por IP/puerto: Restringir acceso basado en direcciones y servicios.
  • Filtrado por horario: Restricciones temporales de acceso.
  • Traducción de direcciones (NAT): Ocultar la estructura de red interna.
  • Inspección profunda de paquetes: Análisis del contenido del tráfico.
  • Logging y auditoría: Registro de actividades para análisis posterior.
iptables -A INPUT -p tcp --dport 22 -j ACCEPT #Permite tráfico SSH entrante en un firewall Linux.
ufw allow 80/tcp #Permite tráfico HTTP en el firewall UFW de Ubuntu.

IDS/IPS

Sistema de Detección de Intrusiones (IDS)

  • Función: Monitorea el tráfico de red o actividad del sistema y alerta sobre posibles intrusiones.
  • Tipos de IDS:
    • NIDS (Network-based): Monitorea tráfico de red.
    • HIDS (Host-based): Monitorea actividad en un host específico.
  • Métodos de detección:
    • Basados en firmas: Detecta patrones conocidos de ataques.
    • Basados en anomalías: Detecta desviaciones del comportamiento normal.
    • Análisis heurístico: Utiliza reglas basadas en experiencia.
  • Ejemplos: Snort (modo IDS), OSSEC, Suricata, Zeek (anteriormente Bro).

Sistema de Prevención de Intrusiones (IPS)

  • Función: Además de detectar, actúa automáticamente ante intrusiones bloqueando el tráfico malicioso.
  • Tipos de IPS:
    • NIPS (Network-based): Actúa sobre tráfico de red.
    • HIPS (Host-based): Actúa sobre actividad en el host.
  • Modos de operación:
    • Modo en línea: Todo el tráfico pasa a través del IPS.
    • Modo pasivo con bloqueo activo: Monitorea pasivamente pero puede bloquear tráfico malicioso.
  • Ejemplos: Snort (modo IPS), Suricata (modo IPS), Cisco FirePOWER, Palo Alto NGFW.

Implementación y gestión

  • Posicionamiento: Puntos estratégicos de la red (perímetro, segmentos críticos).
  • Gestión de alertas: Categorización, priorización y respuesta a alertas.
  • Actualización de firmas: Mantener actualizadas las definiciones de amenazas.
  • Ajuste de falsos positivos: Reducir alertas incorrectas.
  • Integración con SIEM: Centralizar la gestión de eventos de seguridad.
  • Análisis forense: Investigación post-incidentes.
  • Requisitos de rendimiento: Evitar convertirse en cuello de botella.
sudo snort -A console -q -c /etc/snort/snort.conf -i eth0 #Ejecuta Snort en modo IDS mostrando alertas en la consola.
grep "ALERT" /var/log/suricata/fast.log | tail -n 10 #Muestra las últimas 10 alertas registradas por Suricata.

Cifrado y VPN

Cifrado simétrico vs asimétrico

Cifrado simétrico

  • Características: Usa la misma clave para cifrar y descifrar.
  • Ventajas: Rápido, eficiente para grandes volúmenes de datos.
  • Desventajas: Problema de distribución segura de claves.
  • Algoritmos comunes:
    • AES (Advanced Encryption Standard): 128, 192, 256 bits
    • 3DES (Triple DES)
    • ChaCha20
    • Blowfish, Twofish
  • Usos: Cifrado de datos en reposo, comunicaciones donde la clave ya ha sido compartida de forma segura.

Cifrado asimétrico

  • Características: Usa un par de claves (pública y privada).
  • Ventajas: Resuelve el problema de intercambio de claves, proporciona firmas digitales.
  • Desventajas: Más lento, mayor consumo de recursos.
  • Algoritmos comunes:
    • RSA
    • ECC (Elliptic Curve Cryptography)
    • Diffie-Hellman
    • DSA (Digital Signature Algorithm)
    • EdDSA (Ed25519)
  • Usos: Establecimiento de conexiones seguras, firmas digitales, autenticación, PKI.

Sistemas híbridos

  • Funcionamiento: Usa cifrado asimétrico para intercambiar claves simétricas.
  • Ventajas: Combina la seguridad del asimétrico con la velocidad del simétrico.
  • Proceso típico:
    • Generar una clave simétrica aleatoria para la sesión
    • Cifrar la clave de sesión con la clave pública del destinatario
    • Enviar la clave cifrada al destinatario
    • Usar la clave simétrica para el resto de la comunicación
  • Usos: TLS/SSL, PGP, comunicaciones seguras en general.
openssl enc -aes-256-cbc -salt -in archivo.txt -out archivo.enc #Cifra un archivo usando AES-256 (cifrado simétrico).
openssl genrsa -out private.key 2048 #Genera un par de claves RSA de 2048 bits (cifrado asimétrico).

SSL/TLS

Evolución de SSL/TLS

  • SSL 2.0 (1995): Primera versión pública, ya obsoleta y vulnerable.
  • SSL 3.0 (1996): Mejoras significativas, pero también obsoleta (POODLE).
  • TLS 1.0 (1999): Evolución de SSL 3.0, considerada insegura.
  • TLS 1.1 (2006): Protección contra ataques de cifrado por bloques.
  • TLS 1.2 (2008): Mejoras significativas en seguridad, permitiendo algoritmos más fuertes.
  • TLS 1.3 (2018): Versión actual, con handshake más rápido y seguro, menos vulnerabilidades.

Funcionamiento de TLS

  • Handshake TLS: Proceso para establecer una conexión segura.
    • Cliente envía "Client Hello" con versiones y cifrados soportados.
    • Servidor responde con "Server Hello" y su certificado.
    • Cliente verifica el certificado del servidor.
    • Intercambio de claves usando algoritmos asimétricos.
    • Generación de claves de sesión simétricas.
    • Comunicación segura usando claves de sesión.
  • Verificación de certificados: Basada en autoridades de certificación (CA).
  • Perfect Forward Secrecy (PFS): Protección incluso si se compromete la clave privada del servidor.

Implementación y buenas prácticas

  • Certificados: Obtener de CAs confiables, renovar antes de expirar.
  • Configuración segura:
    • Usar solo TLS 1.2+ (deshabilitar SSL y TLS antiguo)
    • Configurar cipher suites seguros
    • Implementar HSTS (HTTP Strict Transport Security)
    • Configurar OCSP Stapling
  • Monitoreo: Verificar regularmente la configuración con herramientas como SSL Labs.
  • Gestión de certificados: Automatizar renovaciones, alertas de expiración.
openssl s_client -connect example.com:443 #Verifica la conexión TLS a un servidor y muestra detalles del certificado.
nmap --script ssl-enum-ciphers -p 443 example.com #Enumera los cipher suites soportados por un servidor HTTPS.

Redes privadas virtuales

Tipos de VPN

  • VPN de acceso remoto: Conecta usuarios individuales a una red privada.
  • VPN site-to-site: Conecta redes completas entre sí.
  • VPN por protocolo:
    • OpenVPN: Flexible, seguro, basado en SSL/TLS
    • IPsec: Robusto, ampliamente compatibile
    • WireGuard: Moderno, ligero, alto rendimiento
    • L2TP/IPsec: Combinación de tunneling y cifrado
    • SSTP: Microsoft, atraviesa firewalls restrictivos
  • VPN por propósito:
    • VPN corporativa: Acceso a recursos empresariales
    • VPN para privacidad: Ocultar actividad en internet
    • VPN para evadir censura: Acceso a contenido bloqueado

Componentes de una VPN

  • Cliente VPN: Software que inicia la conexión desde el dispositivo del usuario.
  • Servidor VPN: Punto de terminación de la conexión VPN.
  • Túnel: Canal de comunicación seguro entre cliente y servidor.
  • Protocolos de túnel: Define cómo se encapsula el tráfico.
  • Protocolos de cifrado: Protegen la confidencialidad de los datos.
  • Autenticación: Verifica la identidad de los participantes.
  • NAT traversal: Permite que la VPN funcione a través de NAT.

Seguridad y consideraciones

  • Ventajas de seguridad:
    • Cifrado del tráfico
    • Ocultamiento de la IP real
    • Protección en redes no confiables
    • Bypass de restricciones geográficas
  • Limitaciones:
    • Posible reducción de velocidad
    • Dependencia del proveedor de VPN
    • No protege contra todos los vectores de ataque
    • Posibilidad de logs de conexión
  • Mejores prácticas:
    • Usar autenticación fuerte
    • Implementar kill switch
    • Actualizar regularmente
    • Verificar política de logs del proveedor
sudo openvpn --config cliente.ovpn #Inicia una conexión VPN usando OpenVPN.
sudo wg-quick up wg0 #Activa una interfaz WireGuard configurada.

Principios de hardening

Hardening de sistemas operativos

  • Actualización del sistema: Mantener todos los componentes actualizados.
  • Principio de mínimo privilegio: Asignar solo los derechos necesarios.
  • Reducción de superficie de ataque: Desinstalar servicios innecesarios.
  • Control de cuentas: Gestionar usuarios, grupos y permisos.
  • Seguridad de arranque: Configurar UEFI Secure Boot, cifrado de disco.
  • Auditoría y logging: Configurar registros detallados de eventos.
  • Firewalls locales: Configurar reglas restrictivas.
  • Virtualización/contenedores: Aislar servicios cuando sea posible.

Hardening de servidores

  • Seguridad física: Controlar acceso a los servidores físicos.
  • Configuración base segura: Partir de una imagen mínima y segura.
  • Gestión segura de acceso remoto: SSH con llaves, 2FA, jumpboxes.
  • Monitoreo de integridad: Detectar cambios no autorizados.
  • Backup y recuperación: Garantizar continuidad de operaciones.
  • Gestión de parches: Proceso definido para actualizar sistemas.
  • Segmentación de red: Separar entornos de producción, desarrollo, etc.
  • Configuración de servicios críticos: Hardening específico para web servers, bases de datos.

Hardening de red

  • Arquitectura de red segura: Diseño con seguridad como prioridad.
  • Segmentación: VLANs, microsegmentación, DMZ.
  • Control de acceso a la red (NAC): Verificación antes de permitir conexiones.
  • Endurecimiento de dispositivos de red: Routers, switches, access points.
  • Gestión segura de dispositivos: Control de acceso administrativo.
  • Protocolos seguros: SSH en lugar de Telnet, HTTPS en lugar de HTTP.
  • Filtrado de tráfico: Firewalls, IDS/IPS, proxy.
  • Monitoreo y análisis de tráfico: Detección de comportamientos anómalos.
sudo lynis audit system #Ejecuta una auditoría de seguridad en el sistema.
sudo chown root:root /etc/shadow && sudo chmod 0400 /etc/shadow #Asegura los permisos del archivo de contraseñas.
sudo systemctl disable unused-service #Desactiva servicios innecesarios para reducir la superficie de ataque.
